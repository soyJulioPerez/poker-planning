# Poker planning

## Descripción

Esta aplicación debe permitir a un equipo, realizar la estimación de H.U mediante Planing Poker

## Herramientas

### Nx Monorepo

Para tener Frontend, Backend en un solo repositorio
Para aprovechar el ecosistema de herramientas

### Frontend

Angular

### Backend

Api Gateway y lambdas

### Persistencia

DynamoDb

## Arquitectura

### Atomic System Design

Quiero armar componentes tipo atomos y moleculas, separados para que puedan ser reusados.

### Poder definir temas

El usuario debe poder elegir de hasta 3 temas preestablecidos
Permitir a los desarrolladores agregar temas posteriormente

### Componentes

Usar Smart y dumbs componentes según se requiera

### Lógica de negocio

Aislar la lógica de negocio en servicios
